(etwas/nichts/jemand/niemand/viel/viele/ein paar/ein bisschen), Länder

Verpackungsangaben.

## Neue Design-Regeln aus dieser Session — im Memory-System gesichert

Drei Regeln sind dauerhaft festgehalten worden.

**Drill-Badges dürfen nicht verraten** (`feedback_drill-badge-verrat.md`)
— Das Cat-Badge an einer Drill-Karte muss vor Lösung unsichtbar sein,
und sein Text darf keine Antwort-Informationen enthalten (keine
Artikel, keine Formen, keine Umlaut-Hinweise, keine Partizipien).
Erlaubt sind reine Kategorie-Namen wie „mask.", „fem.", „Plural",
„schwaches Verb".

**Drill-Karten müssen gemischt sein** (`feedback_drill-karten-mischen.md`)
— Fisher-Yates-Shuffle bei jedem Render, nie vorsortiert nach
Kategorie. Sonst wird der Drill trivial, weil der Lerner am Rhythmus
rät.

**Franks-Drill Design-Pattern** (`feedback_franks-drill-pattern.md`)
— Das Emoji-plus-Artikel-plus-Genus-Farbe-Schema im Chip, Auto-Focus
auf das nächste Item, Live-Feedback beim Tippen sind keine Optionen,
sondern Standard für alle FDxx.

## Commit-Workflow, damit nichts verloren geht

Zwei Repos sind betroffen, je nach Datei. Drill-Dateien liegen im
A1-Subrepo, das Dashboard im B2-Root:

```bash
cd "/Users/frankburkert/Cowork/Projekte/fabDaF/htmlS/A1.1 NEW"
/Users/frankburkert/Cowork/Projekte/fabDaF/scripts/safe-commit.sh \
  "FDxx — Titel" franks-drill/DE_A1_FDxx-*.html

cd "/Users/frankburkert/Cowork/Projekte/fabDaF"
./scripts/safe-commit.sh "dashboard: add FDxx" htmlS/dashboard.html
```

`safe-commit.sh` umgeht die Sandbox-Einschränkungen beim git-Locking.
Niemals mit dem Write-Tool in `.git/` schreiben — das blockiert Cowork
im Hintergrund. Bash ist der einzige sichere Weg. Nach jedem Commit
kurz mit `git rev-parse HEAD origin/main` prüfen, dass beide SHAs
übereinstimmen — der post-commit-Hook pusht automatisch, deshalb nicht
manuell `git push` hinterherschicken.
